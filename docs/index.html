Two surfaces, one workflow

This bundle exists because accessibility problems happen at two different points. Some are created when the interaction is designed. Others are revealed only after implementation. The planner and critic divide that work cleanly.

Planner

/a11y-planner

  • Plans semantic HTML and landmark structure before implementation starts.
  • Maps each interactive pattern to an established APG pattern instead of inventing new behavior ad hoc.
  • Designs keyboard interactions, focus traps, restoration, and tab order explicitly.
  • Specifies how loading, error, invalid, selected, disabled, and success states are announced.
  • Builds the testing strategy so automation and manual checks match the design intent.
Critic

/a11y-critic

  • Reviews the actual implementation for design gaps that automated tooling may not flag.
  • Checks whether semantics match intent, not just whether attributes exist.
  • Looks for incomplete focus management and broken state communication.
  • Requires evidence for major findings instead of speculative lint-style commentary.
  • Applies screen reader, keyboard-only, low-vision, and cognitive lenses before verdict.

Workflow

  1. Run /a11y-planner before implementation when semantics, keyboard behavior, focus logic, and state announcements still need to be decided.
  2. Implement the feature against the plan.
  3. Run /a11y-critic on the resulting code or component.
  4. Run automated and manual accessibility testing.
  5. Refine based on critic evidence and real interaction results.
/a11y-planner ↓ implementation ↓ /a11y-critic ↓ automated + manual testing ↓ refinement

Install

  1. Fast path: install the repo with npx claude-skills.
  2. Manual path: clone the repository and copy root `.claude/skills` and `.claude/agents` into your Claude directories.
  3. Use /a11y-planner before implementation and /a11y-critic after implementation.
npx claude-skills add https://github.com/zivtech/a11y-meta-skills # manual install git clone https://github.com/zivtech/a11y-meta-skills.git cp -r a11y-meta-skills/.claude/skills/* ~/.claude/skills/ cp a11y-meta-skills/.claude/agents/*.md ~/.claude/agents/

What this bundle is trying to prevent

False confidence

  • Attributes exist, but the component still uses the wrong interaction model.
  • Focus technically moves, but does not return where users expect.
  • Error states are visible but not associated with the affected fields.
  • Dynamic content updates without any useful live announcement.

Late discovery

  • Accessibility concerns are postponed until after implementation, when the fix requires a structural rewrite.
  • Testing is treated as a cleanup step instead of part of the design.
  • Teams confuse “keyboard reachable” with “keyboard accessible.”
  • Real users hit friction even though the automation report looks clean.
